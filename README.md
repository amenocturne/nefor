# nefor-team

Командная конфигурация [nefor](https://github.com/amenocturne/nefor) — агентного кодинг-ассистента в терминале.

## Что такое nefor

Nefor — AI-агент для написания кода. Вы описываете задачу на естественном языке,
а nefor планирует, пишет, ревьюит и тестирует код через команду субагентов —
всё прямо в терминале.

Когда вы отправляете сообщение, оно попадает к **lead-оркестратору** — агенту,
который разбивает задачу на шаги и делегирует их **субагентам**:

| Роль             | Что делает                              |
| ---------------- | --------------------------------------- |
| explorer         | Исследует кодовую базу                  |
| builder          | Пишет и редактирует код                 |
| reviewer         | Код-ревью                               |
| tester           | Запускает тесты                         |
| critic           | Критикует план перед выполнением        |
| reflector        | Анализирует решения и компромиссы       |
| prompt-engineer  | Улучшает промпты и инструкции           |

Типичный цикл:

1. Вы описываете задачу
2. Lead отправляет explorer'ов изучить код
3. Lead составляет план и показывает его вам
4. Вы проверяете план и пишете `/approve` (или `/reject` с комментарием)
5. Builder'ы пишут код, reviewer проверяет, tester гоняет тесты
6. Результаты видны в реальном времени

## Пререквизиты

### 1. Homebrew

Если Homebrew не установлен, откройте Терминал и выполните:

```sh
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

После установки следуйте инструкции в выводе, чтобы добавить `brew` в PATH.
Может потребоваться перезапуск терминала.

Проверка:

```sh
brew --version
```

### 2. just

```sh
brew install just
```

### 3. git

macOS поставляется с git. Если нужна свежая версия:

```sh
brew install git
```

### 4. DP CLI

Установить и авторизоваться по
[документации DP](https://devplatform.pages.devplatform.tcsbank.ru/spirit-user-docs/docs/cli/).

## Установка

```sh
git clone https://gitlab.tcsbank.ru/crit-autoloans/nefor nefor-team
cd nefor-team
just install
```

`just install` устанавливает nefor через Homebrew и копирует командную
конфигурацию в `~/.config/nefor/`.

Если конфигурация уже стоит и нужно обновить:

```sh
# Пропустит, если конфиг уже есть
just copy

# Перезатрёт существующий конфиг
just copy force
```

## Запуск

```sh
# Из директории репозитория
just run

# Или из любого места после `just install`
nefor
```

При первом запуске nefor авторизуется через DP, получает Nestor JWT,
загружает список моделей и открывает чат.

## Выбор модели

По умолчанию используется модель, отмеченная `is_default` в Nestor.
Чтобы выбрать другую:

```sh
NEFOR_TEAM_MODEL=tgpt/some-other-model just run
```

## Команды в чате

| Команда    | Что делает                                                |
| ---------- | --------------------------------------------------------- |
| `/approve` | Одобрить предложенный план                                |
| `/reject`  | Отклонить план с пояснением                               |
| `/model`   | Без аргумента — пикер моделей; с аргументом — переключить |
| `/new`     | Новый чат (очищает транскрипт, создаёт новую сессию)      |
| `/resume`  | Восстановить предыдущую сессию                            |
| `/yolo`    | Отключить все запросы разрешений                          |
| `/safe`    | Вернуть запросы разрешений                                |
| `/help`    | Справка                                                   |
| `@путь`    | Прикрепить файл к сообщению (автокомплит)                 |

## Горячие клавиши

| Клавиша       | Действие                                          |
| ------------- | ------------------------------------------------- |
| `Ctrl+O`      | Развернуть/свернуть детали (thinking, tool calls) |
| `Ctrl+B`      | Показать/скрыть сайдбар с графом агентов          |
| `Esc`         | Прервать текущий ход (double-Esc — прервать всё)  |
| `Enter` / `A` | Одобрить инструмент в popup'е разрешений          |
| `D` / `Esc`   | Отклонить инструмент в popup'е                    |

## Разрешения

По умолчанию nefor спрашивает перед выполнением `dispatch-graph` (запуск
агентов) и `bash` (shell-команды). Безопасные read-only bash-команды
авто-одобряются. Остальное — popup с подтверждением.

`/yolo` отключает все запросы. `/safe` возвращает.

## Решение проблем

**Ошибки авторизации / "Nestor JWT exchange failed"** — DP-сессия
истекла. Переавторизуйтесь: `dp auth login`.

**"NEFOR_PLUGIN_DIR is not set"** — вы запускаете Lua-конфиг напрямую.
Используйте `just run` или `nefor`.

**"opts.agentic_loop is required"** — конфиг устарел. Обновите:
`just copy force`.
