# nefor-team

Командная конфигурация [nefor](https://github.com/amenocturne/nefor)

## Пререквизиты

- **DP CLI** — установка по внутренней документации. Ожидается, что `dp` будет в
  `/usr/local/bin/dp`, `~/.nessy/dp_v13.4.2/dp` или `$PATH`

## Установка

```sh
git clone https://gitlab.tcsbank.ru/crit-autoloans/nefor nefor-team
cd nefor-team
just install
```

## Как это работает

Nefor — агентный хост. Задача из чата попадает к **lead-оркестратору** —
высокоуровневому агенту, который планирует и делегирует. Код не пишет. Вместо
этого запускает **субагентов** (explorer, builder, reviewer, tester, и т.д.),
каждый из которых ограничен в доступных инструментах:

| Роль     | Что делает                       | Инструменты                          |
| -------- | -------------------------------- | ------------------------------------ |
| explorer | Исследует кодовую базу           | read, list, search, bash (read-only) |
| builder  | Пишет код                        | read, list, search, write, bash      |
| reviewer | Код-ревью                        | read, list, search                   |
| tester   | Запускает тесты                  | read, list, search, bash             |
| critic   | Критикует план перед выполнением | read, list, search                   |

Типичный цикл: lead отправляет explorer'ов изучить код -> составляет план ->
показывает план пользователю -> после `/approve` запускает builder'ов -> reviewer
проверяет -> tester гоняет тесты.

## Команды в чате

| Команда    | Что делает                                                |
| ---------- | --------------------------------------------------------- |
| `/approve` | Одобрить предложенный план                                |
| `/reject`  | Отклонить план с пояснением                               |
| `/model`   | Без аргумента — пикер моделей; с аргументом — переключить |
| `/new`     | Новый чат (очищает транскрипт, создаёт новую сессию)      |
| `/resume`  | Восстановить предыдущую сессию                            |
| `/yolo`    | Отключить все запросы разрешений (опасно)                 |
| `/safe`    | Вернуть запросы разрешений                                |
| `/help`    | Справка                                                   |
| `@путь`    | Прикрепить файл к сообщению (автокомплит)                 |

## Горячие клавиши

| Клавиша       | Действие                                          |
| ------------- | ------------------------------------------------- |
| `Ctrl+O`      | Развернуть/свернуть детали (thinking, tool calls) |
| `Ctrl+B`      | Показать/скрыть сайдбар с графом агентов          |
| `Esc`         | Прервать текущий ход (double-Esc — прервать всё)  |
| `Enter` / `A` | Одобрить инструмент в popup'е разрешений          |
| `D` / `Esc`   | Отклонить инструмент в popup'е                    |

## Разрешения

По умолчанию nefor спрашивает перед выполнением `dispatch-graph` (запуск агентов)
и `bash` (shell-команды). Безопасные read-only bash-команды авто-одобряются через
`da`. Остальное — popup с подтверждением. `/yolo` отключает все запросы.
