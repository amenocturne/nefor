# Documentation Mindset

When working near or on documentation:

- Documentation is a first-class artifact, not an afterthought.
- Read existing docs before writing code that touches a documented domain. Don't contradict or duplicate what's already written.
- When you change behavior, check if docs describe that behavior — flag if stale.
- Good docs capture _why_, not _what_ — code already shows what.
- Don't duplicate information — link to the source of truth.
- Keep docs high-level — code is the source of truth for details.

Project-specific conventions (folder structure, naming, formatting, tooling) come from the project's AGENTS.md files. Follow whatever the project specifies.
